# Touch me slowly — онлайн дэлгүүр

Тавилга, интерьерийн онлайн дэлгүүр. **GitHub Pages** дээр байршиж, өгөгдлөө **Supabase**-д хадгална.

```
📦 Файлын бүтэц
├── index.html            ← сайт өөрөө
├── supabase-config.js    ← ТАНЫ ЗАСАХ ГАНЦ ФАЙЛ (URL + түлхүүр)
├── .nojekyll
├── assets/               ← script, фонтууд (засах шаардлагагүй)
└── supabase/
    └── schema.sql        ← өгөгдлийн сангийн бүтэц + эхний өгөгдөл
```

---

## 1-р алхам. Supabase тохируулах (~10 минут)

1. **[supabase.com](https://supabase.com)** дээр бүртгүүлнэ (GitHub эрхээрээ нэвтэрч болно) → **New project** дарна.
   - Project name: дурын нэр (ж: `touch-me-slowly`)
   - Database password: аюулгүй нууц үг зохиож **хадгалж авна**
   - Region: `Southeast Asia (Singapore)` — Монголд хамгийн ойр
2. Project бэлэн болмогц зүүн цэснээс **SQL Editor** → **New query** нээнэ.
3. `supabase/schema.sql` файлын агуулгыг **бүхэлд нь** хуулж буулгаад **Run** дарна.
   Ногоон "Success" гарвал хүснэгтүүд, хамгаалалтын дүрэм, демо каталог бүгд үүссэн гэсэн үг.
4. **Админ хэрэглэгч үүсгэх:** зүүн цэс → **Authentication** → **Users** → **Add user** → *Create new user*:
   - Имэйл + нууц үг оруулна (энэ нь админ самбарт нэвтрэх эрх тань болно)
   - ✅ **Auto Confirm User** — заавал идэвхжүүлнэ
5. **Түлхүүрээ авах:** зүүн доод ⚙ **Project Settings** → **API**:
   - `Project URL` (https://xxxx.supabase.co)
   - `Project API keys` хэсгээс **anon / public** түлхүүр
6. Энэ хавтасны **`supabase-config.js`** файлыг нээж, хоёр утгыг өөрийнхөөрөө солино:

```js
window.TMS_CONFIG = {
  SUPABASE_URL: "https://xxxx.supabase.co",
  SUPABASE_ANON_KEY: "eyJhbGciOi...",
};
```

> 💡 anon түлхүүр сайтын кодонд ил байх нь **хэвийн** — Supabase-ийн Row Level Security
> дүрмүүд өгөгдлийг хамгаална. Харин **service_role** түлхүүрийг энд ХЭЗЭЭ Ч бүү тавь.

## 2-р алхам. GitHub Pages дээр байршуулах (~5 минут)

1. **[github.com](https://github.com)** → баруун дээд **+** → **New repository**:
   - Repository name: ж. `touch-me-slowly`
   - **Public** сонгоно (үнэгүй Pages-д шаардлагатай) → **Create repository**
2. Гарч ирэх хуудасны **"uploading an existing file"** холбоос дээр дарна.
3. Энэ хавтасны **бүх файл, хавтсыг** (index.html, supabase-config.js, .nojekyll, assets/, supabase/) цонх руу **чирж оруулаад** → **Commit changes**.
   - Хавтсуудыг бүхэлд нь чирвэл дотоод бүтэц хэвээр хадгалагдана.
4. Repo дотор **Settings** → зүүн цэс **Pages**:
   - Source: **Deploy from a branch**
   - Branch: **main**, хавтас: **/ (root)** → **Save**
5. 1–2 минут хүлээгээд дараах хаягаар сайт тань нээгдэнэ:

```
https://ТАНЫ-НЭР.github.io/touch-me-slowly/
```

> 🌐 Өөрийн домэйн (ж: www.tanaidelguur.mn) холбохыг хүсвэл: Settings → Pages →
> Custom domain хэсэгт домэйноо бичээд, домэйныхоо DNS дээр CNAME бичлэг нэмнэ.

## 3-р алхам. Ашиглах

| Юу | Хаана |
|---|---|
| **Админ нэвтрэх** | Сайтын баруун дээд буланд буй 👤 (Админ) товч → Supabase-д үүсгэсэн имэйл/нууц үгээ оруулна |
| **Бараа, брэнд, нийтлэл, ангилал** | Засвар хиймэгц шууд өгөгдлийн санд хадгалагдана |
| **Контент таб** (нүүр текст, хямдралын цаг) | Өөрчилсний дараа **«Өөрчлөлтийг хадгалах»** товчийг заавал дарна |
| **Зураг оруулах** | Нэвтэрсэн үед сонгосон зураг Supabase Storage-д хуулагдаж, байнгын хаягтай болно |
| **Захиалга** | Худалдан авагч бүр илгээж чадна; жагсаалтыг зөвхөн нэвтэрсэн админ харж, төлөвийг нь шинэчилнэ |

**Демо горим:** `supabase-config.js`-ээ бөглөөгүй байхад сайт хэвийн харагдана,
гэхдээ өгөгдөл хадгалагдахгүй (админ самбарын зүүн доод буланд «Демо горим» гэж заана).

## Түгээмэл асуулт

**Дараа нь өөрчлөлт хийвэл?** GitHub repo дотроо тухайн файлыг нээж ✏️ (Edit) дараад
шинэчилж Commit хийхэд л хангалттай — Pages автоматаар дахин байршуулна.

**Захиалга орж ирэхгүй бол?** Хөтчийн Console (F12) дээр алдаа шалгана; ихэвчлэн
`supabase-config.js`-ийн URL/түлхүүр буруу, эсвэл schema.sql-ээ ажиллуулаагүйгээс болдог.

**Storage-ийн алдаа гарвал?** schema.sql-ийн зургийн сан үүсгэх хэсэг эрхийн улмаас
алдаа өгч болзошгүй — тэр үед Dashboard → **Storage** → **New bucket** → нэр: `images`,
**Public bucket: ON** гэж гараар үүсгэнэ.

**Төлбөр жинхэнээр авах уу?** Одоогоор QPay/SocialPay зэрэг нь захиалгын мэдээлэлд
бүртгэгдэх сонголт төдий. Жинхэнэ төлбөрийн нэгтгэлд банк/ХҮҮ-тэй гэрээ ба сервер
талын код шаардлагатай тул энэ хувилбарт ороогүй.
